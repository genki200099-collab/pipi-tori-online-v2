# ピピとりオンライン 設定組み合わせ点検メモ

## 点検対象設定

- ラウンド数：1〜6
- マッド・ピッグ：あり / なし
- ババブタ失点：任意数値（UI初期値 -50、サーバーでは絶対値ペナルティとして保持）
- 残り手札の失点：1枚ごとに-3点 / カード数字分だけ失点
- 開始時3枚パス：あり / なし
- 開始時ペア捨て：あり / なし
- ピック後ペア処理：ペア候補選択 / スキップ

## 状態遷移の整合性

### 通常開始
`lobby -> playing`

### 開始時3枚パスのみON
`lobby -> passing -> playing`

### 開始時ペア捨てのみON
`lobby -> initialPair -> playing`

### 3枚パス＋開始時ペア捨てON
`lobby -> passing -> initialPair -> playing`

3枚パスはゲーム開始時のみ。第2ラウンド以降の補充時には再実行しない。

## 主要ルールの衝突確認

### 3枚パス
- パス先は時計回りの次プレイヤー。
- ババブタは `passableCardIds` に含めず、サーバー側でも拒否。
- 全員が3枚選択してから同時移動するため、順番による有利不利やカードIDの消失が起きない。
- CPUは通常カードから3枚を自動選択する。

### 開始時ペア捨て
- 3枚パスON時は、パス完了後に開始時ペア捨てへ進む。
- ペア捨ては任意。人間はスキップ可能。
- CPUは進行停止防止のため、可能なペアを自動で捨てる。
- ババブタはペア対象外。

### ピック後ペア処理
- ピック後に同じ数字がある場合、自動では捨てない。
- プレイヤーは候補カードを選ぶか、スキップできる。
- CPUは進行停止防止のため候補があれば自動でペア処理する。

### 失点計算
- `flat3`：残り通常手札1枚につき-3点。
- `faceValue`：残り通常手札は数字分だけ失点。
- マッド・ピッグなし：スペード11は通常カード。
- マッド・ピッグあり＋flat3：スペード11は手札/ごちそう山のどちらでも追加-13点。
- マッド・ピッグあり＋faceValue：スペード11は-40点扱い。ごちそう山にある場合は+1点を数えたうえで-40点。
- ババブタ失点は部屋設定値を使用。

## UI点検

### スマホ
- 部屋作成設定は1列表示。
- 手札は横スクロール。
- 3枚パス選択カードは上に上がる。
- 開始時ペア候補カードはさらに上に上がる。
- ピック/ペア候補は横スクロールで選択可能。
- テーブルは横スクロールして崩れを避ける。

### PC
- 部屋作成設定はグリッド表示。
- プレイヤー欄は広い画面で4列化。
- テーブルは横幅を確保しつつ、必要時のみスクロール。

## 静的チェック結果

- `node --check server.js` 実行対象。
- WebSocketハンドラ存在確認。
- `server.listen` 存在確認。
- 設定値が `publicState` 経由でクライアントへ渡ることを確認。
- 3枚パス、開始時ペア捨て、ピック後ペア、カード重複防止、最弱判定が残っていることを確認。


## 演出強化点検
- 演出はCSSとクライアント表示中心で追加し、サーバーのゲーム判定ロジックは維持。
- 3枚パス、開始時ペア捨て、ピック後ペア選択、最終結果に視覚演出を追加。
- `prefers-reduced-motion` に対応し、端末設定でアニメーション軽減可能。
- WebSocket通信・カード判定・得点計算には影響しない。


## UI冗長性点検・整理
- 詳細設定は部屋作成時だけ必要なため折りたたみへ移動。
- ルールサマリーは参照用のため折りたたみへ移動。
- 進行ログは詳細確認用のため折りたたみへ移動。
- 状態表示は長文説明を廃止し、設定チップに集約。
- 手札上部の説明文は行動に必要な一文へ短縮。
- 演出トーストは常時フェイズ表示ではなく、トリック決着・ピック結果・ペア選択・最終結果中心に抑制。

## 再接続機能点検
- create/join/reconnect時に部屋コード・playerId・名前を保存。
- 切断時、サーバーは席を削除せず `ws=null` として保持。
- reconnectではplayerId優先、次に同名かつ切断中の席へ復帰。
- 既に別WebSocketが残っている場合は古い接続を閉じて新しい接続に差し替える。
- 進行中の部屋でも、保存済みIDまたは同名切断席なら復帰可能。
