# 全体点検・UI改善レポート

作成日時：2026-06-21 15:15 JST

## 修正した不具合

### 状態表示
`passing` と `initialPair` のフェイズ中、上部の状態表示が `-` になっていたため、以下のように表示されるよう修正。

- passing → 3枚パス
- initialPair → 開始ペア

### 最終結果の終了理由
最終結果画面の終了理由が常に「第2ラウンド終了理由」と表示される可能性があったため、実際のラウンド番号を使うよう修正。

### トリック結果表示の重複
ピック専用画面と直前トリック結果バナーで、勝者/最弱情報が重複表示される場面があったため、トリック結果バナーはトリック確認中だけ表示するよう整理。

### 古いWIN/PICK対象表示
次トリックに入っても直前トリックのWIN/PICK対象バッジが残る可能性があったため、トリック確認中だけ表示するよう修正。

### 演出トースト
演出トースト関数が定義されているのに呼び出されていなかったため、重要イベントだけ表示されるよう復帰。
