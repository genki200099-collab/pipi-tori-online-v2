# ピック候補絞りルール追加・全体点検

作成日時：2026-06-21 16:04 JST

## 追加ルール

部屋作成時に「ピック対象数」を選択できる。

- デフォルト：絞らない
- 1〜13枚：最弱側がピック候補をその枚数まで絞る
- 手札枚数が指定数より少ない場合は、手札枚数が候補数になる
- 指定数が手札枚数以上なら実質全手札候補となるため、候補選択フェイズは省略

## 新しい流れ

1. トリック解決
2. 最弱決定
3. ピック対象数が有効で、最弱の手札が指定数より多い場合、最弱側が候補カードを選択
4. 勝者は、その候補カードの中から裏向きで1枚ピック
5. ピック後、従来通りペア選択/スキップ

## CPU対応

- 最弱側がCPUの場合、ババブタ・マッドピッグ・高数字カードを優先して候補に入れる
- 勝者がCPUの場合、候補の中から自動ピック

## 点検・修正

- ピック候補が指定されている場合、勝者は候補外カードを引けないようサーバー側で制御
- 候補カードの中身は勝者側には公開せず、枚数だけ表示
- ピック画面ちらつき対策のDOM再生成抑制を維持
- 最弱判定で非リードスート優先ルールを壊していた古い保険処理を削除
- 再接続・3枚パス・開始時ペア捨て・ペア選択・得点計算・Render Build対策を維持
